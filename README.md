# Прогноз виживання пасажирів «Титаніка» в Excel
Метою проєкту було спробувати побудувати модель прогнозування виживання пасажирів «Титаніка» в Excel, щоб на практиці зрозуміти, як працюють різні підходи до моделювання і з якими труднощами можна зіткнутися без спеціалізованих бібліотек.

---

## Дані
- Відкритий набір даних **Titanic (Kaggle)**
- Цільова змінна: `Survived` (0 — не вижив, 1 — вижив)

## Підготовка даних

Перед побудовою моделей було виконано базову підготовку даних:
- обробка пропущених значень (Age заповнено середнім значенням у межах кожного титулу)
- виділення титулів з імен пасажирів та обʼєднання рідкісних титулів у окрему категорію
- перетворення категоріальних змінних у числові (One-Hot Encoding для Sex, Embarked, Title)
  
---

### 1. Baseline модель
Аналізуючи  дані traine, можна сказили наступне, що вижило 38,24% та загинуло 61,76%
Тому наша початкова гіпотеза: всі пасажири не вижили.
- Kaggle score: ~0.68  
Цей результат використовується як базовий рівень для порівняння.

### 2. Лінійна регресія
Ідея наступна, використовуючи рівняння лінійної регресії та надбудови в Excel  -> Solver  знайти кофіцієнти для наших для нашої моделі  та ьазуючись на цьому зробити прогноз.
Файл line_reg.xlsx

Результат:
- модель повертала значення за межами [0;1], фактично ми знаходимо score кожного пасажира, результат не можна інтерпретувати як ймовірність.

### 3. Логістична регресія
Щоб отримати йомвірність використовуємо сигмоїдну функцію, з оптимізація через log loss.
Файл log-reg.xlsx

Результат:
- частина коефіцієнтів занулялась
- модель працює не коректно всі результати прогнозу більші на 0,5, тобто всі вижили
- З позитивного наші дані прогнозу в межах [0;1], ми отримали ймовірність

### 4. Ручний підбір ознак
Витративши 20 хв часу, спробував підібрати коефіцієнти в ручну для нашої моделі мінімізуючи log loss.
Результат:
- log loss зменшився зменшився на  29%
- результат можелі в межах в межах [0;1]
- модель стала стабільнішою

Фінальний результат:
- Kaggle score: **0.76794**
