# Прогноз виживання пасажирів «Титаніка» в Excel

Метою проєкту було побудувати модель прогнозування виживання пасажирів «Титаніка» **в Excel**, щоб на практиці зрозуміти, як працюють базові підходи до моделювання та з якими обмеженнями можна зіткнутися без використання спеціалізованих ML-бібліотек.

Проєкт виконано в навчальних цілях, з фокусом на логіку моделей, а не на максимальний результат.

---

## Дані
- Відкритий набір даних **Titanic (Kaggle)**
- Цільова змінна: `Survived`  
  - `0` — пасажир не вижив  
  - `1` — пасажир вижив  

---

## Підготовка даних

Перед побудовою моделей було виконано базову підготовку даних:

- обробка пропущених значень (Age заповнено середнім значенням у межах кожного титулу)
- виділення титулів з імен пасажирів та обʼєднання рідкісних титулів у окрему категорію
- перетворення категоріальних змінних у числові  
  (One-Hot Encoding для `Sex`, `Embarked`, `Title`)
- підготовка даних до використання в Excel Solver

---

## 1. Baseline модель

Початковий аналіз тренувальних даних показав:
- вижили: **38.24%**
- загинули: **61.76%**

На основі цього було сформовано просту гіпотезу:  
**усі пасажири не вижили**.

**Результат:**
- Kaggle score: ~**0.68**

Цей результат використовується як базовий рівень для порівняння з іншими моделями.

---

## 2. Лінійна регресія

Наступним кроком була спроба побудувати модель на основі лінійної регресії з використанням **Excel Solver** для пошуку коефіцієнтів.

Файл: `line_reg.xlsx`

**Результат:**
- модель повертала значення за межами діапазону `[0;1]`
- отриманий результат є числовим скором, але **не може інтерпретуватися як ймовірність**
- підхід виявився непридатним для задачі класифікації

![Linear regression result](https://github.com/user-attachments/assets/265e6eeb-0aed-487e-b6a8-ced958245ab9)

---

## 3. Логістична регресія

Для отримання ймовірностей було реалізовано логістичну регресію з використанням сигмоїдної функції та оптимізацією через **log loss**.

Файл: `log-reg.xlsx`

**Результат:**
- частина коефіцієнтів була занулена Solver
- усі прогнозні значення виявилися більшими за `0.5`  
  (модель прогнозує, що всі вижили)
- позитивний момент — значення прогнозу знаходяться в межах `[0;1]` і можуть інтерпретуватися як ймовірності

![Logistic regression result](https://github.com/user-attachments/assets/fa04168b-9af0-47d3-9b55-e67402afaa1a)

---

## 4. Ручний підбір ознак і коефіцієнтів

Щоб краще зрозуміти поведінку моделі, було виконано ручний підбір коефіцієнтів з фокусом на мінімізацію log loss.

**Результат:**
- log loss зменшився приблизно на **29%**
- прогнозні значення залишаються в межах `[0;1]`
- модель стала стабільнішою

**Фінальний результат:**
- Kaggle score: **0.76794**

---

## Висновки

У ході проєкту вдалося:

- зрозуміти різницю між лінійною та логістичною регресією
- на практиці побачити обмеження Excel Solver при роботі з великою кількістю ознак
- переконатися, що підбір ознак і коефіцієнтів суттєво впливає на результат моделі


Побудувати модель прогнозування в Excel можливо, зокрема з використанням логістичної регресії та сигмоїдної функції. Такий підхід дає результат кращий за baseline, однак Excel має суттєві обмеження для оптимізації параметрів. На практиці Excel доцільно використовувати разом з іншими інструментами, які краще підходять для підбору параметрів моделей.
